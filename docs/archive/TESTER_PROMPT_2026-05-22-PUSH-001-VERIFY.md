## What changed

The Rank-1 / P0 defect from `pf-tester-feedback-2026-05-22.md` (PUSH-001) is fixed in code. The server cron dispatcher no longer calls FCM's Legacy HTTP API. It now sends through the **Expo Push API** (`https://exp.host/--/api/v2/push/send`), which accepts the `ExponentPushToken[...]` format the mobile app actually stores. `FCM_SERVER_KEY` is no longer required. PUSH-002 (Android `google-services.json`) is resolved by design under this path.

**Important:** the fix is committed **locally only** (`1879c5b`); it has **not been pushed to `origin/main`** (the automated pass has no GitHub credentials). It must be pushed before an EAS build or staging deploy will include it.
