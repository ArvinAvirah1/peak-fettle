## WHAT WAS BUILT

### TICKET-011 — Transparent AI Plan Reasoning ✅
**Files:** `routes/plans.js` (updated)

- New endpoint: `POST /plans/generate`
- Calls Claude Haiku 4.5 with user history (last 14 days of sets), health metrics (last 7 days), user constraints, and a filtered exercise candidate list
- Returns `{ plan_id, session: { exercises: [...] }, reasoning: "..." }`
- `reasoning` is AI-generated, always cites a specific data point (volume delta, RPE, sleep, HRV, injury flag). Generic copy fails validation.
- New-user fallback: "You're new — this plan will adapt as you log more sessions."
- Paid-tier gate: returns 403 with upgrade copy for free users.
- Plan saved to `plans` table with `is_ai_generated = TRUE`.

### TICKET-012 — Injury & Limitation Constraint Filter ✅
**Files:** `migrations/20260504_user_constraints.sql`, `routes/constraints.js`

- `user_constraints` table: per-user physical limitation flags with canonical type vocabulary (lower_back, knee, shoulder, wrist, hip, ankle, no_barbell, no_jumping, bodyweight_only, custom).
- `exercises.contraindications TEXT[]` column seeded for all existing exercises.
- `GET/POST/DELETE /constraints` routes — upsert-safe, RLS-enforced.
- Plan generation hard-blocks exercises whose `contraindications` overlap with user's active constraint types before the Haiku prompt is built.

### TICKET-013 — Smartwatch Integration (Backend/DB Layer) ✅
**Files:** `migrations/20260504_daily_health_metrics.sql`, `routes/healthMetrics.js`

- `daily_health_metrics` table: stores resting HR, HRV, sleep, active kcal per user per day per source. UNIQUE(user_id, date, source) — upsert-safe.
- `GET /health-metrics` — list user's recent metrics (configurable window, max 365 days).
- `POST /health-metrics` — upsert from HealthKit background sync. COALESCE merge: partial syncs don't clobber complete syncs.
- `GET /health-metrics/summary` — 7-day averages in the shape the AI plan prompt consumes.
- AI plan reasoning already references health metrics when available (e.g. "Your HRV dropped 18% this week — we've reduced intensity").

**Phase D handoff:** Native SwiftUI Apple Watch companion app, Garmin Connect IQ read path, watch-native set logging.

### TICKET-014 — Privacy Architecture Commitment ✅
**Files:** `routes/user.js`

- `GET /user/data-export` — returns all user-owned rows as a downloadable JSON file (workouts, sets, plans, constraints, health metrics, streaks, profile). GDPR Article 20 compliant.
- `DELETE /user/account` — deletes all user data in a DB transaction + prompts auth record deletion. Requires `{ confirm: "DELETE MY ACCOUNT" }` body guard. Rate-limited (3 requests / 15 min).
- TODO in code: wire `supabaseAdmin.auth.admin.deleteUser()` once Supabase service role key is confirmed in environment.

**Remaining for exec/infra:** Update privacy policy with data category list; ship "Your data" screen in React Native app (Phase D frontend).

### Phase E Open Decisions (for exec before kickoff)

- **E-OD-1:** Custom display typeface vs. system fonts. Custom font ≈ +200 KB bundle + licensing cost; significantly stronger brand distinctiveness.
- **E-OD-2:** Animated onboarding brand intro (15–20 s loop). Adds ~2 weeks to Phase E timeline.
- **E-OD-3:** Chart library — Victory Native XL (Skia-based, cross-platform, recommended) vs. custom Skia charts (full brand control, higher build cost). Affects E-004 scope.
- **E-OD-4:** Light mode — spec is dark-first only. Full light mode doubles QA scope. Recommendation: defer to Phase F.

### Phase E Acceptance Criteria (hard gates)

1. Zero hardcoded hex values remain in component files on any platform.
2. All 5 themes render correctly on iOS, Android, and React Native with no artifacts.
3. Theme persists across app restarts and account migrations.
4. WCAG 2.1 AA contrast verified by automated tooling across all 5 themes.
5. All animations respect system Reduce Motion setting.
6. All interactive elements meet 48×48 pt minimum touch target.
7. Design QA sprint closes 100% of P0 visual deltas vs. `peak_fettle_design_spec.docx`.
8. VoiceOver / TalkBack can complete full core journey (log workout → view percentile → change theme) without visual dependency.
